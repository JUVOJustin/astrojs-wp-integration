The package ships a JWT-first Astro auth bridge for WordPress.
It is built on top of `fluent-wp-client` and is meant to be the central place where Astro resolves one authenticated WordPress client or client config per request.

## What It Gives You

- `createWordPressAuthBridge(config)` for login, cookie handling, and request auth resolution
- `loginAction` for username/password -> JWT login
- `getClient(context)` for a ready-to-use `WordPressClient`
- `resolveUser(context)` and `isAuthenticated(context)` for middleware checks
- `getClientConfig(context)` when you need raw client config instead of the client instance

The bridge is client-first, uses web-standard runtime APIs, and does not expose a legacy raw JWT action-auth helper.
It uses the published client's `loginWithJwt()` and `getCurrentUser()` helpers instead of duplicating request handling.

## WordPress Requirement

Enable the [JWT Authentication for WP REST API](https://es.wordpress.org/plugins/jwt-authentication-for-wp-rest-api/) plugin and set `JWT_AUTH_SECRET_KEY` in WordPress.

## Basic Setup

```ts
// src/lib/auth/bridge.ts
import { createWordPressAuthBridge } from 'wp-astrojs-integration';

export const wordPressAuthBridge = createWordPressAuthBridge({
  baseUrl: import.meta.env.WP_URL,
  cookieName: 'wp_user_session',
});
```

## Middleware

Use the bridge as the single place that resolves the current request into one authenticated client.

```ts
// src/middleware.ts
import { defineMiddleware } from 'astro:middleware';
import { wordPressAuthBridge } from './lib/auth/bridge';

export const onRequest = defineMiddleware(async (context, next) => {
  if (context.url.pathname === '/login') {
    return next();
  }

  const wp = await wordPressAuthBridge.getClient(context);

  if (!wp) {
    return Response.redirect(new URL('/login', context.url), 302);
  }

  try {
    context.locals.user = await wp.getCurrentUser();
    context.locals.wp = wp;
  } catch {
    wordPressAuthBridge.clearAuthentication(context.cookies);
    return Response.redirect(new URL('/login', context.url), 302);
  }

  return next();
});
```

## Action Factories with the Bridge Client

Action factories use the bridge client directly, so you do not need to rebuild `baseUrl`, `auth`, or `authHeaders` for each action.

```ts
// src/actions/index.ts
import {
  createCreatePostAction,
  createUpdatePostAction,
  createDeletePostAction,
  pageSchema,
} from 'wp-astrojs-integration';
import { wordPressAuthBridge } from '../lib/auth/bridge';

export const server = {
  login: wordPressAuthBridge.loginAction,

  createPost: createCreatePostAction(wordPressAuthBridge.getClient),
  updatePost: createUpdatePostAction(wordPressAuthBridge.getClient),
  deletePost: createDeletePostAction(wordPressAuthBridge.getClient),

  createPage: createCreatePostAction(wordPressAuthBridge.getClient, {
    resource: 'pages',
    responseSchema: pageSchema,
  }),
};
```

If you prefer static credentials, build one client yourself and pass it directly:

```ts
import { WordPressClient } from 'wp-astrojs-integration';

const wp = new WordPressClient({
  baseUrl: import.meta.env.WP_URL,
  auth: {
    username: import.meta.env.WP_USERNAME,
    password: import.meta.env.WP_APP_PASSWORD,
  },
});

export const server = {
  createPost: createCreatePostAction(wp),
};
```

## Low-Level Escape Hatch

If you need to create the client yourself, the bridge can also return the resolved config.

```ts
import { WordPressClient } from 'wp-astrojs-integration';

const clientConfig = await wordPressAuthBridge.getClientConfig(Astro);

if (!clientConfig) {
  return Astro.redirect('/login');
}

const wp = new WordPressClient(clientConfig);
```

## Custom Auth Resolution

The bridge can also resolve auth from request headers or other request-scoped sources.

```ts
import { createAuthResolver, createWordPressAuthBridge } from 'wp-astrojs-integration';
import type { ActionAPIContext } from 'astro:actions';

export const bridge = createWordPressAuthBridge({
  baseUrl: import.meta.env.WP_URL,
  authResolver: createAuthResolver<ActionAPIContext>((context) => {
    return context.request.headers.get('authorization');
  }),
});
```

## Login Payload Schema

```ts
import { z } from 'astro/zod';
import { wordPressLoginInputSchema } from 'wp-astrojs-integration';

type LoginPayload = z.input<typeof wordPressLoginInputSchema>;
```
