## Setup

Register the three user action factories in your Astro project's `src/actions/index.ts`:

```ts
import {
  createCreateUserAction,
  createUpdateUserAction,
  createDeleteUserAction,
} from 'wp-astrojs-integration';
import { WordPressClient } from 'fluent-wp-client';

const wp = new WordPressClient({
  baseUrl: import.meta.env.WP_URL,
  auth: {
    username: import.meta.env.WP_USERNAME,
    password: import.meta.env.WP_APP_PASSWORD,
  },
});

export const server = {
  createUser: createCreateUserAction(wp),
  updateUser: createUpdateUserAction(wp),
  deleteUser: createDeleteUserAction(wp),
};
```

---

## Creating a user

```ts
const { data, error } = await Astro.callAction(actions.createUser, {
  username: 'editor-jane',
  email: 'jane@example.com',
  password: 'strong-password',
  name: 'Jane Editor',
  roles: ['editor'],
});
```

Required fields are `username`, `email`, and `password`.

---

## Updating a user

`updateUser` accepts a required `id` and any writable user fields.

```ts
const { data, error } = await Astro.callAction(actions.updateUser, {
  id: 15,
  name: 'Jane E.',
  description: 'Editorial lead',
});
```

---

## Deleting a user

WordPress requires user deletion to be force-delete and requires reassignment of existing content.

This action enforces that with a required `reassign` field:

```ts
const { data, error } = await Astro.callAction(actions.deleteUser, {
  id: 15,
  reassign: 1,
  force: true,
});

// data -> { id: 15, deleted: true, reassignedTo: 1 }
```

If `reassign` is omitted, input validation fails before any request reaches WordPress.

## Typed schemas

User actions expose base schemas for extension:

- `createUserInputSchema`
- `updateUserInputSchema`
- `deleteUserInputSchema`

You can extend input validation and response parsing:

```ts
import { WordPressClient } from 'fluent-wp-client';
import { createCreateUserAction, createUserInputSchema } from 'wp-astrojs-integration';
import { z } from 'astro/zod';

const wp = new WordPressClient({
  baseUrl: import.meta.env.WP_URL,
  auth: {
    username: import.meta.env.WP_USERNAME,
    password: import.meta.env.WP_APP_PASSWORD,
  },
});

export const server = {
  createUser: createCreateUserAction(wp, {
    schema: createUserInputSchema.extend({
      app_source: z.string().optional(),
    }),
    responseSchema: z.object({
      id: z.number().int().positive(),
      slug: z.string(),
    }),
  }),
};
```
